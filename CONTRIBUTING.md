# Contributing to PrintOps

Thank you for your interest in contributing.

## Basic Workflow

1. Fork the repository.
2. Create a feature branch.
3. Make your changes.
4. Open a pull request into `main`.

## Expectations

- Keep changes focused.
- Do not commit secrets, passwords, tokens, private keys, or environment files.
- Add documentation for new features when appropriate.
- Make sure security checks pass before requesting review.
